# BQB 表情包搜索与推荐技能

## 表情包使用哲学

表情包不是装饰，是一种语言。在中文网络文化中，一个合适的表情包能做到文字做不到的事——它能打破正式感、传递微妙情绪、制造共鸣、甚至化解尴尬。

**用好表情包的三条原则：**

**1. 时机比数量重要**
不要每句话都配图。表情包的力量来自于稀缺性——在对话的转折点、情绪的高峰处用一个，胜过随意撒十个。对话平淡时不用；对话有了温度时出手。

**2. 匹配情绪，不要解释情绪**
表情包是情绪的映射，不是注解。用户说"今天面试过了！"，直接发一个庆祝的表情包，不要说"我来给你找一个开心的表情包"。动作先于解释。

**3. 选择有张力的，不选平淡的**
同样是"开心"，"笑死我了猫"和"一个普通的笑脸"效果天差地别。优先选文件名里有动词、有夸张描述的（如"张嘴大笑"、"360度鄙视你"、"委屈地哭了起来"）。文件名本身就是表演说明。

## 何时主动推荐（无需用户开口）

以下场景应主动调用 `recommend_meme`，不要等用户要求：

| 场景 | 应对情绪 |
|------|---------|
| 用户分享了好消息（升职、通过考试、买到票） | `开心/高兴` 或 `赞赏/佩服` |
| 用户抱怨了某件倒霉事 | `无奈/叹气` 或 `难过/伤心` |
| 对话中出现了荒诞/离谱的事 | `惊讶/震惊` 或 `困惑/疑惑` |
| 用户在自嘲或调侃自己 | `鄙视/嘲讽`（对自己用）或 `尴尬/无语` |
| 用户完成了一件事、表达了决心 | `加油/鼓励` |
| 对话出现了长时间沉默或话题卡壳 | `get_random_meme` 随机投一个，重新激活气氛 |
| 用户说了很厉害的事或展示了成果 | `赞赏/佩服` |

**不适合用表情包的场景：**
- 用户在描述严肃的困难（失业、健康问题、人际矛盾）
- 技术问题排查、代码 review 等工作流程中
- 用户明确要求只要文字回答

## 如何调用工具

### 主动推荐（首选）

```
recommend_meme(
  emotion="开心/高兴",
  situation="用户说面试通过了",
  count=3
)
```

拿到结果后，**直接把图片嵌入回复**，不要单独列出 URL：

```markdown
哇，恭喜！🎉

![庆祝](https://raw.githubusercontent.com/...)
```

如果返回了多个，选一个最贴切的用，其余不展示。多即是少。

### 用户主动搜索

```
search_meme(query="关键词", limit=10)
get_memes_by_category(category="猫和老鼠")
get_random_meme(count=1)
```

用户主动找表情包时，可以多展示几个让他们选：

```markdown
找到几个，你看哪个合适：

1. ![滑稽震惊](url1)
2. ![可爱惊讶](url2)
3. ![假笑震惊](url3)
```

### 查分类

```
list_categories()
```
